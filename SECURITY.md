### English

#### Security Policy

We take security seriously and strive to ensure that our project is as secure as possible. Please follow the guidelines below if you discover a security vulnerability in this project.

##### 1. **Reporting Security Vulnerabilities**

If you discover a security vulnerability in this project, we ask that you follow these steps to report it:

* **Do not create issues on GitHub**: Please do not create public issues for security vulnerabilities.
* **Email us directly**: Report the vulnerability privately via email to [yuzeguitar@gmail.com](mailto:yuzeguitar@gmail.com).
* Provide a detailed description of the vulnerability, including any relevant steps to reproduce it, and the impact of the issue.
* We will acknowledge your report as soon as possible and work with you to resolve the issue promptly.

##### 2. **Responsible Disclosure**

We ask that security vulnerabilities be disclosed responsibly. Once we have reviewed the vulnerability and developed a fix, we will work with you to coordinate a public disclosure. We appreciate your cooperation in ensuring the security of the project.

##### 3. **Security Updates**

We will make every effort to address security vulnerabilities in a timely manner and release fixes when necessary. Updates will be shared with users and contributors via GitHub releases.

Thank you for helping keep this project secure!

### Deutsch

#### Sicherheitsrichtlinie

Wir nehmen Sicherheit ernst und bemühen uns, unser Projekt so sicher wie möglich zu gestalten. Bitte befolgen Sie die folgenden Richtlinien, wenn Sie eine Sicherheitslücke in diesem Projekt entdecken.

##### 1. **Sicherheitslücken melden**

Wenn Sie eine Sicherheitslücke in diesem Projekt entdecken, bitten wir Sie, die folgenden Schritte zu befolgen, um sie zu melden:

* **Erstellen Sie keine Issues auf GitHub**: Bitte erstellen Sie keine öffentlichen Issues für Sicherheitslücken.
* **Kontaktieren Sie uns direkt per E‑Mail**: Melden Sie die Sicherheitslücke privat per E‑Mail an [yuzeguitar@gmail.com](mailto:yuzeguitar@gmail.com).
* Geben Sie eine detaillierte Beschreibung der Sicherheitslücke an, einschließlich der relevanten Schritte zur Reproduktion und der Auswirkungen des Problems.
* Wir werden Ihre Meldung so schnell wie möglich bestätigen und mit Ihnen zusammenarbeiten, um das Problem zügig zu beheben.

##### 2. **Verantwortungsvolle Offenlegung**

Wir bitten darum, Sicherheitslücken verantwortungsvoll offenzulegen. Sobald wir die Sicherheitslücke geprüft und einen Fix entwickelt haben, werden wir mit Ihnen zusammenarbeiten, um eine öffentliche Offenlegung zu koordinieren. Wir schätzen Ihre Zusammenarbeit, um die Sicherheit des Projekts zu gewährleisten.

##### 3. **Sicherheitsupdates**

Wir werden uns bemühen, Sicherheitslücken zeitnah zu beheben und bei Bedarf Fixes zu veröffentlichen. Updates werden über GitHub Releases mit Benutzern und Mitwirkenden geteilt.

Vielen Dank, dass Sie dazu beitragen, dieses Projekt sicher zu halten!
