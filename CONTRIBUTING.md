### English

#### Contributing

Thank you for considering contributing to this project! We welcome contributions from everyone. To ensure a smooth and positive experience for everyone, please take a moment to review the following guidelines.

##### 1. **Reporting Issues**

If you find a bug or have a feature request, please open an issue in the [Issues section](https://github.com/yuzeguitarist/Windsurf-Reset/issues). Be sure to include as much detail as possible, such as error messages, steps to reproduce the issue, and any relevant logs.

##### 2. **Submitting Pull Requests**

* Fork the repository and create a new branch for your feature or bug fix.
* Write clear, concise commit messages describing your changes.
* Test your changes before submitting the pull request.
* Follow the existing code style and conventions used in the project.
* Submit the pull request to the `main` branch.
* Make sure to include a description of your changes in the PR.

##### 3. **Code of Conduct**

By contributing to this project, you agree to follow our [Code of Conduct](https://github.com/yuzeguitarist/Windsurf-Reset/blob/main/CODE_OF_CONDUCT.md).

##### 4. **Licensing**

By submitting a pull request, you agree that your contributions are licensed under the project's [license](https://github.com/yuzeguitarist/Windsurf-Reset/blob/main/LICENSE).

#### 贡献指南

感谢您考虑为本项目做出贡献！我们欢迎每个人的贡献。为确保每个人都能有一个顺利且积极的体验，请花一些时间查看以下准则。

##### 1. **报告问题**

如果您发现了 bug 或有功能请求，请在 [Issues 部分](https://github.com/yuzeguitarist/Windsurf-Reset/issues) 提交问题。请尽可能提供详细的信息，如错误信息、重现步骤和相关日志等。

##### 2. **提交 Pull Request**

* Fork 本仓库并为您的功能或 bug 修复创建一个新分支。
* 编写简洁明了的提交信息，描述您的更改。
* 提交 pull request 前，请先测试您的更改。
* 遵循项目中现有的代码风格和约定。
* 提交 pull request 到 `main` 分支。
* 在 PR 中包含您更改的描述。

##### 3. **行为准则**

通过为本项目贡献，您同意遵守我们的 [行为准则](https://github.com/yuzeguitarist/Windsurf-Reset/blob/main/CODE_OF_CONDUCT.md)。

##### 4. **许可协议**

通过提交 pull request，您同意您的贡献遵循项目的 [许可证](https://github.com/yuzeguitarist/Windsurf-Reset/blob/main/LICENSE)。

### Deutsch

#### Mitwirkung

Vielen Dank, dass Sie in Erwägung ziehen, zu diesem Projekt beizutragen! Wir begrüßen Beiträge von allen. Um eine reibungslose und positive Erfahrung für alle zu gewährleisten, nehmen Sie sich bitte einen Moment Zeit, um die folgenden Richtlinien zu überprüfen.

##### 1. **Fehler melden**

Wenn Sie einen Fehler finden oder eine Funktionsanforderung haben, öffnen Sie bitte ein Issue im [Issues-Bereich](https://github.com/yuzeguitarist/Windsurf-Reset/issues). Stellen Sie sicher, dass Sie so viele Details wie möglich angeben, wie z. B. Fehlermeldungen, Schritte zur Reproduktion des Fehlers und relevante Logs.

##### 2. **Pull-Requests einreichen**

* Forken Sie das Repository und erstellen Sie einen neuen Branch für Ihr Feature oder Bugfix.
* Schreiben Sie klare, prägnante Commit-Nachrichten, die Ihre Änderungen beschreiben.
* Testen Sie Ihre Änderungen, bevor Sie den Pull-Request einreichen.
* Befolgen Sie den bestehenden Code-Stil und die Konventionen des Projekts.
* Reichen Sie den Pull-Request an den `main`-Branch ein.
* Stellen Sie sicher, dass Sie eine Beschreibung Ihrer Änderungen im PR hinzufügen.

##### 3. **Verhaltenskodex**

Indem Sie zu diesem Projekt beitragen, erklären Sie sich mit unserem [Verhaltenskodex](https://github.com/yuzeguitarist/Windsurf-Reset/blob/main/CODE_OF_CONDUCT.md) einverstanden.

##### 4. **Lizenzierung**

Indem Sie einen Pull-Request einreichen, erklären Sie sich damit einverstanden, dass Ihre Beiträge unter der [Lizenz](https://github.com/yuzeguitarist/Windsurf-Reset/blob/main/LICENSE) des Projekts lizenziert sind.
